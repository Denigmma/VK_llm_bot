# VK LLM Bot

💬 Умный VK-бот для диалогов с LLM-моделями через OpenRouter.

Бот умеет вести несколько независимых чатов внутри VK, хранит историю в SQLite, помогает настраивать каждый чат под отдельную задачу и поддерживает не только текст, но и изображения. Его можно использовать как персонального ассистента, учебного помощника, мультимодальный чат для решения задач по фото и удобную оболочку над несколькими моделями OpenRouter прямо в VK.

## ✨ Что умеет бот

- Вести несколько чатов на одного пользователя с независимой историей.
- Хранить контекст сообщений в чате, настройки и вложения в SQLite.
- Интерфейс команд в сообщениях VK.
- Переключать модель, профиль ключа и reasoning для каждого чата отдельно.
- Автоматически выбирать `free` или `pay` ключ по выбранной модели.
- Принимать фотографии с подписью и отправлять их в Client как мультимодальный запрос.

## 🧠 Как это работает

У каждого пользователя есть набор чатов. У каждого чата свои:

- название;
- модель;
- профиль OpenRouter: `free` или `pay`;
- настройки reasoning;
- системный промпт;
- история сообщений.

Когда пользователь пишет обычное сообщение, бот берет активный чат, достает последние сообщения из истории и отправляет их в OpenRouter.

Когда пользователь отправляет фото с подписью, бот:

1. достает текст подписи;
2. извлекает из VK самый большой `photo.size.url`;
3. сохраняет и текст, и ссылку на изображение;
4. отправляет это в OpenRouter как мультимодальный `content` из `text + image_url`.

Если текущая модель, скорее всего, текстовая и не умеет работать с картинками, бот временно переключает запрос на vision-модель внутри того же профиля ключа.

## 🎨 Как выглядит общение с ботом

Бот старается отвечать не сухим сплошным текстом, а более структурированно. Например:

- `/help` разбит на блоки вроде `💬 Чаты`, `⚙️ Настройки`, `🧠 Другое`;
- `/chats` показывает номера и названия чатов;
- `/new` запускает мини-мастер настройки нового чата;
- `/models` показывает список моделей с пометками `free/pay` и знанием о vision/text-профиле;
- `/settings` выводит настройки активного чата в удобном читаемом виде.

## 🚀 Быстрый старт

### 1. Создайте `.env`

Возьмите `.env.example` как основу:

```env
VK_GROUP_TOKEN="..."
VK_CONFIRMATION_CODE="..."
VK_SECRET_KEY="..."

OPENROUTER_BASE_URL="https://openrouter.ai/api/v1"

API_KEY_OPENROUTER="..."
MODEL="openai/gpt-5.4"

FREE_API_KEY_OPENROUTER="..."
FREE_MODEL="nvidia/nemotron-3-super-120b-a12b:free"

DATABASE_URL="sqlite:///./bot.db"

DEFAULT_REASONING_ENABLED="false"
DEFAULT_REASONING_EFFORT="medium"
DEFAULT_MAX_CONTEXT_MESSAGES="20"
DEFAULT_TEMPERATURE="0.7"

APP_HOST="0.0.0.0"
APP_PORT="8000"
```

Что откуда брать:

- `VK_GROUP_TOKEN`, `VK_CONFIRMATION_CODE`, `VK_SECRET_KEY` — из настроек VK Callback API.
- `API_KEY_OPENROUTER` — платный ключ OpenRouter.
- `FREE_API_KEY_OPENROUTER` — бесплатный ключ OpenRouter.
- `MODEL` — модель по умолчанию для платного профиля.
- `FREE_MODEL` — модель по умолчанию для бесплатного профиля.

### 2. Установите зависимости

```bash
python -m venv venv
venv\Scripts\pip install -r requirements.txt
```

### 3. Запустите бота

```bash
python run.py
```

или так:

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

После первого запуска рядом с проектом создастся `bot.db`, если используется стандартный `DATABASE_URL`.

## 🌍 Настройка VK Callback API

Запустите ngrok:

```bash
ngrok http 8000
```

В настройках VK Callback API укажите URL такого вида:

```text
https://xxxx.ngrok-free.app/vk/callback
```

Поведение callback endpoint:

- `confirmation` возвращает ровно `VK_CONFIRMATION_CODE`;
- остальные события проверяются по `secret`;
- при неверном `secret` бот ничего не обрабатывает и возвращает `ok`;
- обработка сообщений уходит в `BackgroundTasks`, поэтому VK быстро получает ответ.

## 👤 Как пользоваться ботом

### Создание нового чата

Команда:

```text
/new
```

После этого бот пошагово проведет через настройку:

1. попросит ввести название чата;
2. предложит выбрать модель;
3. предложит настроить reasoning;
4. сохранит чат и сделает его активным.

На любом шаге можно отправить:

```text
/skip
```

если текущее значение устраивает.

### Переключение между чатами

Сначала посмотрите список:

```text
/chats
```

Потом переключитесь на нужный чат:

```text
/chat 2
```

В списке показываются именно названия чатов, чтобы ими было удобно пользоваться в реальной работе.

### Удаление ненужного чата

```text
/deletechat 2
```

Бот удалит выбранный чат по номеру из списка `/chats`.

### Настройка модели

```text
/models
/model 1
/model openai/gpt-5.4
/model nvidia/nemotron-3-super-120b-a12b:free
```

Если в `model_id` есть `:free`, бот автоматически использует `FREE_API_KEY_OPENROUTER`.
Если `:free` нет, используется `API_KEY_OPENROUTER`.

### Настройка reasoning

```text
/reasoning
/reasoning off
/reasoning on
/reasoning low
/reasoning medium
/reasoning high
```

## 🖼️ Работа с фотографиями

Бот поддерживает сценарий:

- пользователь отправляет фото;
- добавляет подпись, например: `реши это задание` или `что изображено на картинке?`;
- бот отправляет и текст, и фото в модель.

Пример пользовательского сценария:

1. Вы отправляете фотографию листка с задачей.
2. В подписи пишете: `реши задачу и объясни по шагам`.
3. Бот передает это в OpenRouter как мультимодальный запрос.
4. Если текущая модель не подходит для изображений, бот использует vision fallback.


## 🔑 Логика free/pay профилей

У бота есть два профиля OpenRouter:

- `free`
- `pay`

Они нужны, чтобы можно было удобно использовать бесплатные и платные модели в одном боте.

Автологика:

- модель с `:free` -> используется `free` профиль;
- модель без `:free` -> используется `pay` профиль.

Ручное переключение тоже доступно:

```text
/key
/key free
/key pay
```

При ручном переключении бот устанавливает профиль и дефолтную модель из `.env`.

## 🧾 Команды

```text
/help — показать справку.
/new — создать новый чат и пройти настройку.
/chats — показать последние 10 чатов.
/chat 1 — переключиться на чат по номеру.
/deletechat 1 — удалить чат по номеру.
/reset — очистить историю текущего чата.
/settings — показать настройки текущего чата.
/models — показать список предустановленных моделей.
/model — показать текущую модель.
/model 1 — выбрать модель по номеру.
/model model_id — установить модель вручную.
/key — показать текущий профиль ключа.
/key free — переключиться на free профиль.
/key pay — переключиться на pay профиль.
/reasoning — показать настройки reasoning.
/reasoning off — отключить reasoning.
/reasoning on — включить reasoning.
/reasoning low|medium|high — выбрать уровень reasoning.
/system — показать системный промпт.
/system текст — обновить системный промпт.
/skip — пропустить текущий шаг настройки нового чата.
```

## 🛠️ Что реализовано внутри

- FastAPI webhook для VK Callback API;
- фоновые задачи через `BackgroundTasks`;
- SQLite + SQLAlchemy для хранения пользователей, чатов и сообщений;
- хранение ссылок на изображения и attachment metadata;
- интеграцию с OpenRouter Chat Completions API;
- мультимодальные запросы `text + image_url`;
- автопереключение между `free/pay` ключами;
- эвристику и fallback для vision-моделей;
- пошаговую настройку новых чатов;
- удаление чатов;
- улучшенный текстовый UX для команд и настроек;
- логирование запросов и ответов OpenRouter в терминал сервера.

## 📋 Логи и диагностика

Во время работы в терминале сервера видно:

- входящие события VK;
- сколько фото пришло в сообщении;
- какая модель реально использовалась;
- был ли применен vision fallback;
- статус ответа OpenRouter;
- предупреждения о нестандартных ответах OpenRouter.
