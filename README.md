# VK LLM Bot

💬 VK-бот для удобной работы с LLM-моделями через OpenRouter прямо в привычном чате.

Проект превращает диалог в VK в полноценный рабочий интерфейс для текстовых и мультимодальных моделей: с отдельными чатами под разные задачи, гибкой настройкой модели, free/pay-профилями, контекстом, регенерацией ответов и поддержкой фотографий.

## ✨ Возможности

- Несколько независимых чатов на одного пользователя.
- Отдельные настройки для каждого чата: модель, ключевой профиль, reasoning, system prompt, длина контекста.
- Хранение истории, настроек и вложений в SQLite.
- Пошаговая настройка нового чата через `/new`.
- Автоматическое переключение между `free` и `pay` ключами по выбранной модели.
- Поддержка фотографий: подпись + изображение отправляются в OpenRouter как мультимодальный запрос.
- Vision fallback, если текущая модель, вероятно, не умеет работать с изображениями.
- Управление контекстом: просмотр, изменение длины, обрезка последних или самых ранних пользовательских сообщений.
- Регенерация последнего ответа и регенерация с правкой последнего промпта.
- Более дружелюбный формат ответов: `/help`, `/settings`, `/chats` и другие команды выводятся структурированно и читаемо.

## 🧠 Как устроен диалог

У каждого пользователя есть набор чатов. У каждого чата свои:

- название;
- модель;
- профиль OpenRouter: `free` или `pay`;
- настройки reasoning;
- системный промпт;
- длина контекста;
- история сообщений.

Когда пользователь пишет обычное сообщение, бот берет активный чат, достает нужную часть истории и отправляет ее в OpenRouter.

Важно: длина контекста теперь считается по пользовательским сообщениям, а не по сырым строкам `user/assistant` в базе. Поэтому если пользователь отправил 3 своих запроса, `/context` покажет `3/20`.

## 🖼️ Работа с фотографиями

Бот поддерживает мультимодальный сценарий:

1. пользователь отправляет фото;
2. добавляет подпись, например `что изображено на фото?` или `реши задачу с листка`;
3. бот достает самый большой `photo.size.url` из VK attachment;
4. сохраняет и подпись, и ссылку на фото в БД;
5. отправляет в OpenRouter `text + image_url`.

Если текущая модель, скорее всего, текстовая, бот временно использует vision-модель внутри того же профиля ключа и сообщает об этом в ответе.

## 🚀 Быстрый старт

### 1. Создайте `.env`

Возьмите `.env.example` как основу:

```env
VK_GROUP_TOKEN="..."
VK_CONFIRMATION_CODE="..."
VK_SECRET_KEY="..."

OPENROUTER_BASE_URL="https://openrouter.ai/api/v1"

API_KEY_OPENROUTER="..."
MODEL="openai/gpt-5.4"

FREE_API_KEY_OPENROUTER="..."
FREE_MODEL="nvidia/nemotron-3-super-120b-a12b:free"

DATABASE_URL="sqlite:///./bot.db"

DEFAULT_REASONING_ENABLED="false"
DEFAULT_REASONING_EFFORT="medium"
DEFAULT_MAX_CONTEXT_MESSAGES="20"
DEFAULT_TEMPERATURE="0.7"

APP_HOST="0.0.0.0"
APP_PORT="8000"
```

Что откуда брать:

- `VK_GROUP_TOKEN`, `VK_CONFIRMATION_CODE`, `VK_SECRET_KEY` — из настроек VK Callback API.
- `API_KEY_OPENROUTER` — платный ключ OpenRouter.
- `FREE_API_KEY_OPENROUTER` — бесплатный ключ OpenRouter.
- `MODEL` — модель по умолчанию для платного профиля.
- `FREE_MODEL` — модель по умолчанию для бесплатного профиля.

### 2. Установите зависимости

```bash
python -m venv venv
venv\Scripts\pip install -r requirements.txt
```

### 3. Запустите бота

```bash
python run.py
```

или:

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

После первого запуска рядом с проектом создастся `bot.db`, если используется стандартный `DATABASE_URL`.

## 🌍 Настройка VK Callback API

Запустите ngrok:

```bash
ngrok http 8000
```

В настройках VK Callback API укажите URL такого вида:

```text
https://xxxx.ngrok-free.app/vk/callback
```

Что делает endpoint:

- `confirmation` возвращает ровно `VK_CONFIRMATION_CODE`;
- остальные события проверяются по `secret`;
- при неверном `secret` бот игнорирует событие и возвращает `ok`;
- обработка сообщений идет через `BackgroundTasks`, поэтому webhook быстро отвечает VK.

## 👤 Как пользоваться ботом

### 1. Создать новый чат

```text
/new
```

После этого бот предложит:

1. ввести название чата;
2. выбрать модель;
3. настроить reasoning;
4. завершить создание чата.

На любом шаге можно отправить:

```text
/skip
```

### 2. Переключаться между чатами

```text
/chats
/chat 2
```

В списке `/chats` показываются именно названия чатов, поэтому между ними удобно переключаться даже при одинаковых моделях.

### 3. Удалять ненужные чаты

```text
/deletechat 2
```

### 4. Менять модель и профиль ключа

```text
/models
/model 1
/model openai/gpt-5.4
/model nvidia/nemotron-3-super-120b-a12b:free
/key
/key free
/key pay
```

Логика такая:

- если у модели есть `:free`, бот автоматически использует `FREE_API_KEY_OPENROUTER`;
- если `:free` нет, используется `API_KEY_OPENROUTER`.

### 5. Настраивать reasoning

```text
/reasoning
/reasoning off
/reasoning on
/reasoning low
/reasoning medium
/reasoning high
```

### 6. Управлять контекстом

```text
/context
/context 30
/context trim head 3
/context trim tail 2
```

Что это значит:

- `/context` — показывает, сколько пользовательских сообщений сейчас попадает в контекст, например `3/20`;
- `/context 30` — меняет длину контекста для текущего чата;
- `/context trim head 3` — удаляет 3 последних пользовательских запроса вместе с ответами;
- `/context trim tail 2` — удаляет 2 самых ранних пользовательских запроса вместе с ответами.

### 7. Перегенерировать последний ответ

```text
/regen
```

Бот удалит из БД последний пользовательский промпт и последний ответ модели, затем снова отправит этот же промпт в модель с прежним контекстом.

### 8. Перегенерировать с правкой промпта

```text
/regen Сформулируй короче и по пунктам
```

В этом режиме бот:

1. берет последний пользовательский промпт;
2. удаляет старую пару `промпт + ответ`;
3. использует ваш новый текст;
4. отправляет обновленный промпт в модель с тем же предыдущим контекстом.

Если у исходного промпта была фотография, она сохраняется и повторно отправляется вместе с отредактированным текстом.

## 🧾 Команды

```text
/help — показать справку.
/new — создать новый чат и пройти настройку.
/chats — показать последние 10 чатов.
/chat 1 — переключиться на чат по номеру.
/deletechat 1 — удалить чат по номеру.
/reset — очистить историю текущего чата.
/settings — показать настройки текущего чата.
/models — показать список предустановленных моделей.
/model — показать текущую модель.
/model 1 — выбрать модель по номеру.
/model model_id — установить модель вручную.
/key — показать текущий профиль ключа.
/key free — переключиться на free профиль.
/key pay — переключиться на pay профиль.
/reasoning — показать настройки reasoning.
/reasoning off — отключить reasoning.
/reasoning on — включить reasoning.
/reasoning low|medium|high — выбрать уровень reasoning.
/context — показать текущее использование контекста.
/context N — изменить длину контекста.
/context trim head N — удалить последние пользовательские сообщения вместе с ответами.
/context trim tail N — удалить самые ранние пользовательские сообщения вместе с ответами.
/regen — перегенерировать последний ответ.
/regen новый текст — перегенерировать с правкой последнего промпта.
/system — показать системный промпт.
/system текст — обновить системный промпт.
/skip — пропустить текущий шаг настройки нового чата.
```

## 🛠️ Что реализовано внутри

- FastAPI webhook для VK Callback API;
- фоновые задачи через `BackgroundTasks`;
- SQLite + SQLAlchemy для хранения пользователей, чатов и сообщений;
- хранение ссылок на изображения и attachment metadata;
- интеграция с OpenRouter Chat Completions API;
- мультимодальные запросы `text + image_url`;
- автопереключение между `free/pay` ключами;
- эвристика и fallback для vision-моделей;
- пошаговая настройка новых чатов;
- удаление чатов;
- управление контекстом и регенерацией;
- улучшенный текстовый UX для команд и настроек;
- логирование запросов и ответов OpenRouter в терминал сервера.

## 📋 Логи и диагностика

Во время работы в терминале сервера видно:

- входящие события VK;
- сколько фото пришло в сообщении;
- какая модель реально использовалась;
- был ли применен vision fallback;
- статус ответа OpenRouter;
- предупреждения о нестандартных ответах OpenRouter.
